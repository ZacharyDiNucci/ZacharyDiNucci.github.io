About Me

I am a 4th year GIMM major at Boise State and have been learning how to make and program games and other software within the degree. I enjoy all types of games and playing them is what inspire me to develope. My focus within the degree has been learning to program in C# for Unity and C++ and blueprints for Unreal. 

I have lived in Boise my entire life and have had an interest in video games from a young age. I remember my family renting the Gamecube and Xbox from our local Hastings. This is what got me hooked on games and my love would only grow. Around Junior High I saved up my money for my own Gaming PC which my father and I built. This is around the same time I became interested in learning to make my own games through Minecraft Modding. A little ways down you will see the fruits of this journey and what I have created in GIMM.

Portfolio

Demo Reel

This is a Demo reel of some of the works I have done this last semester

Robo Beatdown Unreal

This game was the second game I worked on in Unreal Engine it was inspired by games like SAS zombie assault. I created all the assets in blender and used this game to learn the blueprint system as much as I could. 

Food Flingers

In GIMM every student must make a VR game. Mine was this food fight game which was inspired by the Index Controllers which were what I used to create the throwing of the food using hand gestures. early on you also see a class system where you customise your character by choosing classes they are in.

Pig

This game was created for a group project for a series of minigames where we would switch roles in the creation. For This game I was the artist and so had to use blender to create the assets and art. I also helped come up with the idea for the game and the initial pigs state machine code.

Quack Attack

This was the first Unreal engine game and last game I made with the series of minigames. For this game I created the mechanics for the targets so they would godown when shot and the UI for score. For this one we went simple because a limit in time and also learning from the next game about being to ambitious.

Multiplayer bomber

This game was the third minigame my group created and was a PUN2 multiplayer game. Unfortunatly we were unable to get it working all the way but we used it as a lesson on multiplayer dev. That was to make the game with multiplayer in mind and to not pull in assets form other areas that are not made with that. This was because we built this off our second game which wasn't multiplayer and pulled some asseets however those assets are what ended up breaking the most.

Boise Schools AR

This final project was a collaboration with teachers in the Boise School District in order to teach students various basic skills that they would need in the world. I did the coding for this project and the place holder assets which are shown in the video.
